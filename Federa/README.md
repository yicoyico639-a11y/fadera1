# Federa - 去中心化联邦学习激励协议

一个基于零知识证明的联邦学习任务市场，让数据提供者和训练者在不可信环境中协作训练AI模型，并按贡献自动获得代币奖励。

## 项目概述

Federa将联邦学习改造为一个**利益绑定、自动审计、贡献可验证的开放市场**：任务发起者质押奖金，训练节点质押参与，通过链上梯度验证和贡献评分，实现无需信任的全球AI协作训练。

## 核心创新点

- **可验证训练证明（PoT）**: 训练节点在本地执行梯度计算，并用RISC Zero zkVM生成ZK证明，杜绝虚假梯度。
- **沙普利值贡献度分账**: 通过轻量级近似沙普利值算法，精确量化每个节点对模型性能提升的边际贡献。
- **隐私保护数据注册**: 数据提供者可将数据哈希与访问策略上链，确保原始数据不出域。
- **动态聚合与仲裁**: 聚合节点收集梯度更新并合并为全局模型，若聚合者作恶，其质押将被罚没。
- **模型进化NFT**: 全局模型的每个版本被铸造成一个状态NFT，记录贡献者列表、性能指标和版本血缘。

## 防作恶机制

### 多层防御体系

1. **ZK证明层**: 数学保证梯度更新的计算过程正确性
2. **质押经济层**: 利益绑定，作恶即罚没
3. **统计检测层**: 异常发现（梯度范数检测、余弦相似度检测等）
4. **仲裁挑战层**: 社会共识，独立重算验证
5. **信誉系统层**: 历史约束，长期作恶者被排除

## 架构组件

- **FederaRegistry**: 全局注册表，管理训练节点、聚合节点和数据提供者
- **FederaTaskFactory**: 任务工厂，创建和管理训练任务
- **FederaTask**: 单个训练任务合约，处理训练流程
- **FederaDataRegistry**: 数据注册合约，管理数据集访问权限
- **FederaModelNFT**: 模型NFT合约，记录模型版本和贡献者
- **Trainer Client**: 训练节点客户端，负责模型训练和ZK证明生成
- **Aggregator Service**: 聚合节点服务，负责梯度聚合和贡献计算

## 安装和使用

### 环境要求
- Node.js >= 16
- Python >= 3.8
- Rust (用于RISC Zero集成)

### 安装依赖

```bash
# 安装合约依赖
npm install

# 安装Python依赖
pip install torch numpy scipy
```

### 编译和部署

```bash
# 编译合约
npx hardhat compile

# 运行测试
npx hardhat test

# 部署到本地网络
npx hardhat run scripts/deploy.js --network localhost
```

### 运行训练节点

```bash
# 运行训练节点客户端
cd src/trainer
python trainer_client.py
```

### 运行聚合节点

```bash
# 运行聚合节点服务
cd src/aggregator
python aggregator_service.py
```

## 技术栈

- **区块链**: Ethereum L2 (Base/Arbitrum)
- **智能合约**: Solidity 0.8.x + Hardhat
- **ZK证明**: RISC Zero zkVM (计划集成)
- **训练框架**: PyTorch
- **链下计算**: Python + RISC Zero Guest
- **存储**: IPFS + Filecoin
- **前端**: Next.js + wagmi + viem
- **聚合服务**: Rust
